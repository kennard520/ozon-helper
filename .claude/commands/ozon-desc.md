# /ozon-desc — Ozon 商品描述体检与改写

按 [ozon/card-requirements.md §2](../../ozon/card-requirements.md) 的官方规则，对 Ozon 商品描述做合规体检 + 优化重写。
详细步骤见 [.claude/skills/ozon-desc/SKILL.md](../skills/ozon-desc/SKILL.md)。

---

## 用户输入

```
$ARGUMENTS
```

---

## 你的任务

### 第 1 步：解析参数

第一段非 `键=值` 的内容是**待检查的描述文本**（可多行、可含俄/中/英）。其余按键值对解析：

| 键 | 用途 |
|---|---|
| 文件 / file | 从文件读描述（覆盖正文输入） |
| 链接 / url | 抓 Ozon 友商描述做对照（用 WebFetch） |
| 类目 / category | 影响"必含信息清单"细化 |
| 模式 / mode | `仅检查` / `改写` / `双语对照`（默认双语对照）|
| 源语言 / lang | `zh` / `en` / `ru`（默认自动识别）|

### 第 2 步：检查必填

如果既没有描述正文也没有 `文件=` / `链接=`，回复使用提示（见末尾）。

### 第 3 步：执行 skill

按 [.claude/skills/ozon-desc/SKILL.md](../skills/ozon-desc/SKILL.md) 7 步流程执行：

1. 读 [ozon/card-requirements.md §2](../../ozon/card-requirements.md) 拿规则
2. 识别源语言 + 字符/段落统计（必要时先翻成俄语再评）
3. 5 项"优秀标准"逐项打分（0/1/2，总分 10）
4. 3 大雷区扫描（信息不完整 / 机翻不正确 / 尺寸列举误导）+ 附加扫描（价格 / 联系方式 / 营销禁词 / Aliexpress 套话）
5. 输出体检报告：总分 + 风险等级（绿 8-10 / 黄 5-7 / 红 0-4）+ 五项标准表 + 雷区扫描 + 修改建议
6. 重写（默认开启，除非 `模式=仅检查`），按 §2.7 实操骨架；含尺寸列举时**先反问用户拆 SKU**
7. 末尾给"一键复用"提示

### 第 4 步：硬约束

- **不编造**原稿没有的参数（材质/尺寸/容量）——抓不到写 `[待补]`
- 机翻判定**给具体证据**（哪一句、为什么不地道），不要光说"机翻味重"
- 发现 `Размер: S, M, L` 这类多 SKU 列举 → 先反问用户，不要默认选一档
- 不引入禁用营销词（`скидка / акция / лучший / хит продаж / №1` 等）
- 字数控制 500-1500，4-6 段，含编号或项目符号
