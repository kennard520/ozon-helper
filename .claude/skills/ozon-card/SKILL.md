# Ozon 商品卡片生成 Skill

## 用途

输入一个商品链接，自动产出一份**符合 Ozon 2026 卡片规则**的草稿，包括：

1. 俄语标题（结构化 + 过禁用词检查）
2. 俄语描述（必含要素齐全 + 段落分明）
3. 属性表（通用必填 + 类目相关必填）
4. 图片清单（哪些需要 PS / 哪些要重拍 / 缺哪些角度）
5. 合规自查报告（按 [ozon/card-requirements.md §9](../../../ozon/card-requirements.md) 逐项 check）

## 调用格式

```
/ozon-card https://detail.1688.com/offer/123456789.html
/ozon-card https://item.taobao.com/item.htm?id=xxx
/ozon-card https://detail.1688.com/offer/xxx.html 类目=家居装饰
/ozon-card https://detail.1688.com/offer/xxx.html 已知重量=120 颜色=白/黑
```

可选追加参数：
- `类目=家居装饰|收纳整理|宠物配件|...`
- `重量=Xg`（如果链接抓不到）
- `颜色=...`（多色用 `/` 分隔）
- `品牌=...`（默认 `Нет бренда`）

## 执行步骤

### Step 1: 读取规则源

**先**读取 [ozon/card-requirements.md](../../../ozon/card-requirements.md)，这是本 skill 的规则单一来源。
后续每一步都引用本文件的章节号（如 §1.4 禁用词）。

### Step 2: 抓取商品信息

用 WebFetch 抓取用户给的链接，提取：

- **中文标题**
- **价格**（参考用，不是必须）
- **重量 / 尺寸**（详情页规格表）
- **材质 / 成分**
- **颜色 / 规格选项**
- **主图 URL 列表**
- **详情页文案**（用于描述生成）

抓不到的字段标 `[待补]`，不要瞎编。

### Step 3: 推断类目

如果用户没给类目，按 1688 标题关键词匹配：

| 关键词 | 类目 |
|---|---|
| 宠物/猫/狗/鱼 | 宠物配件 |
| 收纳/整理/储物/折叠 | 收纳整理 |
| 文具/笔/本/办公 | 办公文具 |
| 装饰/摆件/挂件/花瓶 | 家居装饰 |
| 圣诞/万圣/节日/派对 | 节日礼品 |
| 定制/刻字/个性化 | 定制类 |
| 其它 | 通用 |

提示用户："已用 `{推断的类目}`，要换可加 `类目=X`"。

### Step 4: 生成俄语标题（应用规则 §1）

1. **查 §1.1 表选公式**：按推断类目挑对应的命名公式
   - 通用/家居/收纳/厨具/装饰/文具/节日 → `Бренд + Тип + Модель + 关键参数`
   - 服装/鞋类 → `Тип + Бренд + 系列`（顺序反过来）
   - 美妆/健康 → `Бренд + 商品名 + Тип + 规格`
   - 电子 → `Бренд + Тип + Модель + 关键参数（颜色/接口/容量）`
   - 食品/补剂 → `Бренд + Тип + 风味 + 重量 × 件数`
   - 宠物 → `Бренд + Тип + 尺码 + 颜色`
   - 汽配/工具 → `Бренд + Тип + 型号 + 适配车型`
2. **不要把所有特征都堆进去**——只挑该类目的"关键参数"。其余去描述里讲。
3. 翻译为地道俄语（不要直译，不要音译/拉丁拼俄）
4. **跑禁用词表**（§1.4）：命中即换近义中性词
5. **字符自查**（§1.2）：
   - 禁用 `!` `?` `™` `©` `®` `[ ]` `=` `\` `«` `»` emoji 装饰符
   - 不允许全大写（品牌官方拼法除外）
6. **长度自查**（§1.3）：单词 ≤ 27、总长 ≤ 200、推荐 40-120
7. **Model name 字段单独输出**（§1.5）：只填型号/系列代码，**不要**写类型/品牌/颜色/尺寸/件数

例（家居类，本仓主战场）：
- 中文："2026新款北欧风家居装饰玻璃花瓶透明摆件"
- 俄语标题：`Нет бренда Ваза стеклянная прозрачная 25 см для декора в скандинавском стиле`
- Model name 字段：留空（无系列名）
- 长度 79 ✅ 无禁用词 ✅ 单词最长 16 ✅ 公式：品牌(无) + 类型(Ваза) + 关键参数(стеклянная/прозрачная/25 см/скандинавский) ✅

如果**不确定**俄语表达，给 2 个候选让用户挑，不要硬选。

### Step 5: 生成俄语描述（应用规则 §2）

模板（含 §2.1 全部必含项）：

```
[一句话核心卖点 — 用途/场景]

Характеристики:
• Материал: [材质]
• Размер: [尺寸 cm]
• Вес: [重量 g]
• Цвет: [颜色]
• Объём / Ёмкость: [容量，如适用]

Особенности:
• [核心卖点 1]
• [核心卖点 2]
• [核心卖点 3]

Способ применения:
[使用方法/场景说明]

В комплекте:
• [包装清单]
```

**过滤**（§2.2）：联系方式、店铺名、广告词、夸大宣传。

### Step 6: 生成属性表（应用规则 §5）

输出一个表，分两部分：通用必填 + 类目相关。

| 字段 | 值 | 备注 |
|---|---|---|
| Категория | `{类目俄语}` | 用 Ozon 后台最深层级 |
| Бренд | `Нет бренда`（或用户指定） | |
| Модель (Model name) | `{型号/系列}` 或留空 | **不含**类型/品牌/颜色/尺寸/件数 (§1.5) |
| Цвет | `{颜色}` | 用 Ozon 标准色名 |
| Материал | `{材质}` | |
| Вес нетто | `{重量}g` | |
| Габариты | `{长}×{宽}×{高} см` | |
| 包装尺寸 | `[待补]` | 通常+2cm 缓冲 |
| ...（类目相关字段） | | |

未知字段保留 `[待补]`，标黄提示用户填充。

### Step 7: 图片清单（应用规则 §3）

输出"建议的 6+ 张图清单"，每张写**用途 + 拍/PS 要求**：

| # | 类型 | 来源建议 | 处理要求 |
|---|---|---|---|
| 1 | 主图 | 1688 现有主图 | 改纯白底 RGB(255,255,255)、去中文水印、改 3:4 或 1:1、放大到 ≥ 2000×2000 |
| 2 | 细节图 | 1688 细节图 | 去中文标签 |
| 3 | 尺寸标注图 | 1688 或自制 | 文字必须俄语 |
| 4 | 场景图 | 1688 场景图或换 | 贴近俄罗斯家居风格 |
| 5 | 对比图/规格图 | 自制 | 俄语标注 |
| 6 | 包装图 | 1688 包装图 | 去中文 Logo |

如果 1688 主图明显是 AI 合成（光影违和、手指畸形），警告"§3.2 禁止 AI 主图，需自拍或换货源"。

### Step 8: 合规自查报告

按 [ozon/card-requirements.md §9](../../../ozon/card-requirements.md) 表逐项打勾或标红：

```
✅ 主图纯白底
⚠️ 主图含中文水印 → 需 PS
✅ 标题命中类目公式（§1.1）
✅ 标题无禁用词 / 无 !? / 无 ™©® / 无 emoji
✅ 标题非全大写、无音译俄语
✅ 单词最长 ≤ 27 字符，总长 ≤ 200 字符
✅ Model name 字段不含类型/品牌/颜色
✅ 描述含全部 5 项必含要素
❌ 类目相关属性缺 3 项 → 见 Step 6 [待补]
✅ 图片数 ≥ 6
⚠️ 当前 1688 主图疑似 AI 生成 → §3.2 风险
```

## 注意事项

1. **不许瞎编**：链接抓不到的字段标 `[待补]`，让用户补；不要根据"通常"或"一般"凭空填。
2. **俄语翻译**：用专业翻译思路（地道、SEO 友好），不是机械直译。如果不确定，给两个候选让用户选。
3. **规则源单一**：所有规则都从 ozon/card-requirements.md 读，**不要在 SKILL.md 里复制规则**——文档更新时只改一处。
4. **类目细分**：本 skill 给的是大类，最终细分类目要用户在 Ozon 后台用"建议类目"功能验证（1247 个子类目，超出 LLM 记忆范围）。
5. **图片处理**：本 skill 只生成"处理要求清单"，不实际改图。改图用 Photoshop / nano-banana 等。
