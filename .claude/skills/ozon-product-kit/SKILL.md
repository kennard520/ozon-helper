# ozon-product-kit — Ozon 商品卡一键生成 Skill

## 目标

本 skill 是 Ozon 商品卡总控工作流。用户只需要提供以下任意一种输入：

- 一张或多张产品图
- 一个商品链接
- 一段简单中文说明
- 产品图 + 少量参数
- 商品链接 + 少量补充说明

本 skill 自动完成：

1. 识别商品和图片中的可见信息
2. 提取图片里的尺寸、参数、卖点、型号、材质、包装清单等文字信息
3. 区分“已确认 / 图片识别待核对 / 待确认”
4. 设计 10 张 Ozon 商品卡图片方案
5. 调用 `gen-image` skill 逐张生成图片
6. 生成俄语 Ozon 标题、简介、卖点、商品描述、营销文本
7. 生成 30 个以内俄语主题标签
8. 生成 Ozon 富文本内容结构
9. 输出完整路径、文案和复核清单

# 极简输入模式

用户可以只发一句话，例如：

```text
Ozon一键生成：
这张图是三脚架自拍杆，参数按图里识别，生成整套 Ozon 商品卡。
```

处理原则：

1. 不要求用户写长提示词。
2. 自动从图片和文字中提取商品信息。
3. 图片里能看清的尺寸、参数、卖点要提取出来。
4. 识别不清的内容不能当作确定信息。
5. 缺少真实产品图时，不生成冒充实物的主图。
6. 缺少参数时不中断流程，但要标记“待确认”。

# 支持的输入模式

## 1. 产品图模式

用户上传一张或多张产品图。

处理方式：

1. 识别商品类型、颜色、结构、可见配件、可见文字。
2. 对图片进行视觉分析和文字提取。
3. 提取图片中出现的尺寸、参数、卖点、型号、材质、包装清单、安装步骤等。
4. 将清晰可见且合理的信息列入“已确认信息”。
5. 将模糊、不完整、有遮挡或可能识别错误的信息列入“图片识别待核对”。
6. 将图片无法确认的信息列入“待确认信息”。
7. 生成图片时优先使用 `edit` 模式，并把产品图作为参考图。
8. 不要用纯 `create` 重新画产品主图。
9. 不要虚构图片中不存在的配件、功能、品牌和参数。

## 2. 链接模式

用户提供商品链接。

处理方式：

1. 尝试读取链接内容。
2. 提取商品标题、主图、详情图、参数、包装清单、卖点。
3. 如果页面有多张图，优先选择最清晰、最接近商品白底图的图片作为产品参考图。
4. 如果链接无法访问、需要登录、验证码或反爬，则提示用户上传产品图或截图。
5. 链接中没有明确写出的参数不能当作确定信息。
6. 不要复制竞品品牌、店铺名、水印、公司名和侵权词。

## 3. 简单文字模式

用户只提供一句话，例如：

```text
280KG 防水磁力锁，生成一套 Ozon 商品卡。
```

处理方式：

1. 根据文字生成商品理解和图片方案。
2. 缺少产品图时，只能生成方案、文案和场景示意图。
3. 主图、细节图、尺寸图必须提醒用户需要真实产品图。
4. 不要生成“看起来像真实产品”的主图来冒充真实商品图。
5. 所有无法确认的信息写入“待确认”。

## 4. 图 + 参数模式

这是推荐模式。

用户提供产品图，同时补充参数，例如：

```text
商品：黑色三脚架自拍杆
参数：120cm，24cm收纳，7节伸缩，1/4螺丝，铝合金+ABS
要求：生成10张Ozon商品图、俄语listing、标签、富文本
```

处理方式：

1. 产品图作为外观参考。
2. 用户参数作为已确认信息。
3. 图片中文字和参数作为补充来源。
4. 根据商品类型自动设计 10 张图。
5. 调用 `gen-image` 逐张出图。
6. 生成俄语商品资料。

# 图片文字 / 尺寸 / 卖点提取规则

这是本 skill 的关键规则。

## 必须从产品图中提取的信息

如果图片中出现以下内容，必须尝试提取：

- 商品名称
- 型号
- 尺寸，例如长度、宽度、高度、厚度、直径
- 重量
- 材质
- 颜色
- 节数、档位、数量
- 功率、电压、电流、电池容量
- 接口类型，例如 Type-C、USB、1/4 螺丝
- 防水、防尘、耐高温、承重等功能卖点
- 适用设备
- 适用场景
- 安装步骤
- 包装清单
- 警示说明
- 认证标志

## 提取结果必须分级

提取后生成 `image_extracted_info.md`，结构如下：

```markdown
# 图片识别信息

## 图片来源

## 可见商品结构

## 图片文字提取表

| 原图文字 | 识别结果 | 中文解释 | 俄语可用表达 | 可信度 | 使用状态 | 备注 |
|---|---|---|---|---|---|---|

## 已确认信息

## 图片识别待核对

## 明确禁止复制的信息

## 待确认信息
```

## 可信度规则

### 高可信，可作为已确认信息

满足全部条件才可作为已确认信息：

1. 图片中文字清晰可读。
2. 数字和单位完整，例如 `120cm`、`24cm`、`207g`。
3. 与商品外观和类别不冲突。
4. 不是价格、联系方式、二维码、店铺名、水印、竞品品牌。

### 中可信，作为“图片识别待核对”

以下情况不能直接写成确定卖点：

1. 字体模糊。
2. 被遮挡。
3. 数字不完整。
4. 单位不清楚。
5. 可能是图片背景里的无关文字。
6. 参数与商品外观看起来冲突。

### 低可信，不能使用

以下内容必须丢弃：

1. 价格。
2. 折扣。
3. 联系方式。
4. 店铺名。
5. 公司名。
6. 水印。
7. 二维码。
8. 平台促销标签。
9. 竞品品牌。
10. 识别明显错误的乱码。

## 图片中文字如何使用

1. 可以把中文参数翻译成俄语 listing。
2. 不要把中文直接放到最终 Ozon 文案里。
3. 不要让 AI 在生成图里重新绘制中文。
4. 尺寸、参数、卖点可以用于文字内容，但必须注明来源是图片识别。
5. 对重要参数，最终输出中要提示人工复核。

示例：

如果图片中写着：

```text
120cm / 24cm / 7节伸缩 / 1/4螺丝 / 207g
```

可以在 `image_extracted_info.md` 中写：

```markdown
已从图片中识别到：展开高度 120cm、收纳长度 24cm、7 节伸缩、1/4 螺丝接口、约 207g。建议人工复核后用于最终上架参数。
```

俄语 listing 可写：

```text
Высота в разложенном виде — до 120 см, длина в сложенном виде — около 24 см, 7 секций, универсальный винт 1/4".
```

但生成图片时不要让 AI 直接写这些数字。

# 信息可信度分级

所有商品信息必须分为四类。

## 用户确认信息

用户明确提供的信息。优先级最高。

## 图片识别信息

从产品图文字、参数表、包装图或说明图中识别的信息。

图片识别信息可以使用，但重要参数需要在最终结果中提示人工复核。

## 合理推断信息

根据外观和商品类别可以合理判断的信息，例如：

- 黑色外壳
- 三脚架结构
- 可伸缩结构
- 玻璃门安装场景
- 桌面使用场景
- 便携收纳场景

合理推断信息可以谨慎写入描述，但不要写成绝对承诺。

## 待确认信息

以下信息如果用户、链接、图片都没有明确提供，不得编造：

- 净重
- 尺寸
- 材质
- 承重
- 功率
- 电池容量
- 防水等级
- 认证
- 包装清单
- 是否含遥控器
- 是否含手机夹
- 是否含安装螺丝
- 是否支持某具体型号
- 质保时间
- 品牌授权

待确认信息必须在最终结果中单独列出。

# Ozon 图片硬性要求

所有商品图必须遵守：

1. 单张独立图片，不要拼宫格。
2. 竖版 3:4。
3. 推荐尺寸：`1024x1536`。
4. 背景白色或浅色。
5. 白色或透明商品可使用深色背景。
6. 服装、鞋类、配饰类目可使用灰色背景 `#f2f3f5`。
7. 图片清晰，不要模糊。
8. 不要黑白照片。
9. 不要价格。
10. 不要折扣。
11. 不要联系方式。
12. 不要链接。
13. 不要二维码。
14. 不要购买引导，例如“购买”“立即下单”“联系卖家”。
15. 不要中文。
16. 不要水印。
17. 不要品牌 logo，除非用户明确拥有品牌并要求保留。
18. 不要公司名、工厂名、店铺名。
19. 不要侵权商标。
20. 主图不要大段文字。
21. 主图商品不要贴边，四周保留安全边距。
22. 重要细节不要靠近四角，避免被平台徽章遮挡。
23. 不要虚构配件。
24. 不要虚构功能。
25. 不要让 AI 自动生成复杂俄文和参数数字。

---

# 图片文字规则

AI 图片生成不适合直接生成准确俄文、中文、尺寸数字和参数表。

因此：

1. 主图不放文字。
2. 尺寸图只生成干净底图和预留排版区域。
3. 参数图只生成底图，不让 AI 写参数。
4. 富文本底图可以预留标题区、参数区、卖点区，但不要让 AI 写具体文字。
5. 俄文标题、参数、卖点应在后期设计工具中单独排版。
6. 如果用户要求直接出带文字图片，必须提醒：AI 可能生成错误文字，建议后期排版。

# 产品一致性规则

如果用户提供产品图，默认所有图片都使用 `edit` 模式。

必须尽量保持：

- 产品颜色
- 产品比例
- 产品结构
- 接口位置
- 螺丝孔
- 按钮
- 线材
- 支架
- 表面纹理
- 材质质感
- 配件数量

禁止：

- 把产品画成不同型号
- 添加不存在的品牌
- 添加不存在的配件
- 改变接口类型
- 改变产品颜色
- 改变核心结构
- 把产品夸张变形
- 把尺寸比例画得明显不合理

# 总工作流程

## Step 1：解析输入

读取用户输入中的：

- 商品链接
- 产品图片路径
- 商品中文名
- 商品型号
- 已知参数
- 图片中可见文字
- 颜色
- 材质
- 适用场景
- 包装清单
- 禁止内容
- 目标平台
- 目标语言

默认：

- 目标平台：Ozon
- 目标语言：俄语
- 图片尺寸：`1024x1536`
- 图片比例：3:4
- 图片数量：10 张
- 出图方式：优先 `edit`

## Step 2：提取图片信息 image_extracted_info.md

对所有用户上传图片做视觉识别和文字提取。

必须包含：

```markdown
# 图片识别信息

## 产品外观识别

## 可见文字和参数

## 尺寸/规格/卖点提取

## 禁止复制的信息

## 图片识别待核对

## 可用于 Listing 的信息
```

## Step 3：生成商品简报 product_brief.md

必须包含：

```markdown
# 商品简报

## 商品中文名

## 商品俄语名

## 商品类型

## 核心用途

## 用户确认信息

## 图片识别信息

## 合理推断信息

## 待确认信息

## 禁止写入内容

## Ozon 风险检查

## 建议类目
```

注意：

- 不确定类目可以给建议，但不能假装已经确定。
- 涉及带电、液体、食品、儿童、医疗、危险品等，需要标记风险。
- 涉及品牌词，需要提示是否有授权。

## Step 4：设计 10 张商品图 image_plan.md

默认设计 10 张单独图片。

1. 主图
2. 完整展示图
3. 核心细节图
4. 尺寸参数底图
5. 功能卖点图
6. 安装 / 使用步骤图
7. 使用场景图
8. 材质 / 工艺 / 耐用性图
9. 包装 / 配件清单图
10. 富文本首屏底图

每张图必须包含：

- 图片编号
- 图片名称
- 用途
- 推荐模式：create / edit / mask
- 是否使用产品参考图
- 英文生成提示词
- 禁止事项
- 输出文件名

## Step 5：为每张图生成英文提示词

每张图提示词必须包含：

- 商品名称
- 产品外观要求
- Ozon 风格
- 3:4 vertical image
- 1024x1536
- white or light background
- no Chinese text
- no watermark
- no logo
- no QR code
- no price
- no contact information
- keep product consistent with reference image

通用提示词尾部必须加入：

```text
3:4 vertical ecommerce product image, 1024x1536. Clean Ozon marketplace style. No Chinese text, no watermark, no logo, no QR code, no price, no discount, no contact information, no website link, no call-to-action. Keep the product appearance consistent with the reference image. Do not invent accessories or functions.
```

对于信息图底图，额外加入：

```text
Create a clean layout base only. Leave empty space for later Russian text overlay. Do not generate readable text, numbers, labels or parameter tables.
```

## Step 6：调用 gen-image 出图

### 默认 edit 模式

只要用户提供产品图，默认使用：

```powershell
python .claude/skills/gen-image/generate.py "<PROMPT>" --mode edit --image "<PRODUCT_IMAGE_PATH>" --size 1024x1536 --n 1 --name "<IMAGE_NAME>"
```

### create 模式

只有以下情况可使用：

- 用户没有产品图，只做概念图或场景示意图
- 背景氛围图
- 不要求产品 100% 一致

命令：

```powershell
python .claude/skills/gen-image/generate.py "<PROMPT>" --mode create --size 1024x1536 --n 1 --name "<IMAGE_NAME>"
```

### mask 模式

以下情况使用：

- 只去中文
- 只去水印
- 只换背景
- 只改局部
- 只修产品瑕疵
- 只替换某一块区域

命令：

```powershell
python .claude/skills/gen-image/generate.py "<PROMPT>" --mode mask --image "<SOURCE_IMAGE_PATH>" --mask "<MASK_IMAGE_PATH>" --size 1024x1536 --n 1 --name "<IMAGE_NAME>"
```

### 多图合成

如果有产品图和场景图，使用多个 `--image`：

```powershell
python .claude/skills/gen-image/generate.py "<PROMPT>" --mode edit --image "<PRODUCT_IMAGE_PATH>" --image "<SCENE_IMAGE_PATH>" --size 1024x1536 --n 1 --name "<IMAGE_NAME>"
```

## Step 7：读取 _last.json

每次调用 `gen-image` 后，读取：

```text
.claude/skills/gen-image/_last.json
```

记录：

- `mode`
- `model`
- `prompt`
- `image_in`
- `size`
- `n`
- `images[].path`
- `images[].bytes`
- `tokens.total`
- `ts`

不要从 stdout 解析图片路径。以 `_last.json` 为准。

## Step 8：图片质量检查

每张图生成后必须检查：

### 基础合规

- 是否是单张独立图片
- 是否 3:4
- 是否清晰
- 是否不是黑白照片
- 是否背景干净
- 是否没有中文
- 是否没有价格
- 是否没有联系方式
- 是否没有二维码
- 是否没有链接
- 是否没有购买引导
- 是否没有水印
- 是否没有品牌侵权词

### 产品一致性

- 产品颜色是否一致
- 产品结构是否一致
- 产品比例是否明显错误
- 接口、按钮、支架、线材是否被画错
- 是否出现不存在的配件
- 是否出现不存在的功能
- 是否有明显变形

### Ozon 主图检查

主图额外检查：

- 是否无文字
- 是否白底或浅底
- 是否产品居中
- 是否四周留白
- 是否重要部分没有靠近角落

如果图片失败，记录失败原因，重新生成。每张图最多重试 2 次。

# 禁止行为

不得：

1. 把 10 张图拼成一张。
2. 把中文放进图片。
3. 把中文写进俄语 listing。
4. 编造产品参数。
5. 编造包装配件。
6. 编造防水、承重、认证。
7. 使用竞品品牌词。
8. 保留原图水印。
9. 保留原图公司名。
10. 生成价格、折扣、联系方式、二维码。
11. 直接复制竞品详情页文案。
12. 把示意图当成真实主图。
13. 产品图严重变形仍交付。
14. 忽略待确认信息。
15. 输出结果时只说“完成”而不列路径。

# 执行原则

优先级从高到低：

1. Ozon 合规
2. 产品真实性
3. 图片信息可追溯
4. 不虚构参数
5. 图片单张独立
6. 俄语可用
7. 富文本结构清晰
8. 出图效率

如果合规和美观冲突，以合规为准。
如果真实性和氛围感冲突，以真实性为准。
如果图片识别和用户确认冲突，以用户确认信息为准，并在报告中标注差异。
如果参数不确定，以“待确认”为准，不要编造。
