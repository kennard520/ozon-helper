# Ozon 商品描述检查与优化 Skill

## 用途

补 `ozon-card` 之外的"描述专项"工序：用户已经有一份俄语 Описание（自己写的 / 翻译软件出的 / 友商抄的），让我按官方标准体检 + 改稿。**不用先抓 1688 链接**——直接对文本工作。

## 调用格式

```
/ozon-desc <描述俄文，可多行>
/ozon-desc 文件=path/to/desc.txt
/ozon-desc 链接=https://www.ozon.ru/product/...        # 抓友商描述做对照
/ozon-desc <描述> 类目=美发工具 模式=改写              # 直接给改写稿
/ozon-desc <描述> 模式=仅检查                          # 只出报告不改稿
```

参数：
- `类目=...`：影响"必含信息清单"的细节（如美妆要讲肤质；家居讲风格/房间）
- `模式=仅检查 | 改写 | 双语对照`（默认双语对照——给中文/俄文重写稿）
- `源语言=zh | en | ru`（默认自动识别）

## 执行步骤

### Step 1: 读规则源

读 [ozon/card-requirements.md §2](../../../ozon/card-requirements.md)。本 skill 所有判定都引用 §2 的章节号，**不在本文件复制规则**。

### Step 2: 识别源语言 + 字符体检

- 检测语言（俄/中/英/混）
- 字符计数（推荐 500-2000）
- 段落数（≥3 才算"结构良好"）

如果用户给的是中文/英文，**不要直接评分**——先翻成俄语再评，否则等于在评一份不会真上的稿。

### Step 3: 5 项"优秀标准"打分（§2.1）

对每项给 0 / 1 / 2 分（0 = 完全缺失 / 1 = 部分满足 / 2 = 满足），总分 10。

| 项 | 判定要点 |
|---|---|
| ① 与商品相符 | 是否出现"通用套话"或"看起来在讲别的品类"。理想：开头一句就能让人猜出是什么 SKU |
| ② 描述特性和优点 | 至少 2 个**可验证**卖点（材质/工艺/差异化），不是"高品质""优质" |
| ③ 书写得当 | 抽查 3 处：标点空格、词性格变、句子是否通顺。机翻味重要扣分 |
| ④ 信息齐 | 6 类信息（用途 / 材质 / 规格 / 卖点 / 使用方法 / 包装清单）覆盖几项 |
| ⑤ 结构良好 | 至少 3 段 + 有项目符号或编号 |

### Step 4: 3 大雷区扫描（§2.2）

| 雷区 | 检测方式 |
|---|---|
| A. 信息不完整 | 用 Step 3 的 ④ 得分；若 ≤1 → 触发 A |
| B. 机翻不正确 | 抓特征：`100% Brand new`、`высокое качество` 单独成句、英语语序、动词错位、不存在的搭配 |
| C. 尺寸/规格列举 | 正则查：`Размер[:：]\s*[SMLXL,/、\s]+`、`Цвет[:：]\s*\w+[,/、]\s*\w+`，列出多值即触发 |

附加扫描（§2.4 禁项）：
- **商品价格**：`\d+\s*(?:руб|₽|RUB|元|￥)`
- 联系方式：电话、邮箱、URL、`@\w+`、微信号、`Telegram`、`WhatsApp`
- 禁用营销词（同 §1.4）：`скидка / акция / лучший / хит продаж / новинка(作宣传) / №1`
- 套话：`из-за того, что компьютеры отображают цвета`、`Измеряется вручную, допускается погрешность`、`thanks for your understanding`、`100% brand new`

### Step 5: 输出体检报告

```markdown
# Ozon 描述体检 — {产品代号}

## 总分：X / 10  ·  风险等级：{绿 / 黄 / 红}

- 绿（8-10）：可直接上架，仅小优化
- 黄（5-7）：建议改写后上架
- 红（0-4）：会被审核或转化率拖累，必须重写

## 五项标准

| # | 标准 | 得分 | 证据 / 问题 |
|---|---|---|---|
| ① | 与商品相符 | x/2 | ... |
| ② | 特性和优点 | x/2 | ... |
| ③ | 书写得当 | x/2 | 列具体语法/标点错误位置 |
| ④ | 信息齐 | x/2 | 缺失的字段：用途 / 卖点 / 使用方法 / 包装清单 |
| ⑤ | 结构良好 | x/2 | 段数 X / 有无编号 |

## 雷区扫描

- [✅/⚠️/❌] A. 信息不完整
- [✅/⚠️/❌] B. 机翻痕迹    → 列出可疑句子
- [✅/⚠️/❌] C. 尺寸/规格列举 → 列出具体位置
- [✅/⚠️/❌] 含商品价格
- [✅/⚠️/❌] 联系方式 / 营销禁词 / Aliexpress 套话

## 修改建议（按优先级）

1. **必改**：{❌ 项的具体改法}
2. **建议改**：{⚠️ 项}
3. **可选优化**：{加分项，例如 "可以补一句目标用户场景"}
```

### Step 6: 重写（默认开启，除非 `模式=仅检查`）

按 [§2.7 实操骨架](../../../ozon/card-requirements.md) 重写：

```
[开头 1-2 句] 是什么 + 给谁 + 解决什么

[详述段] 材质 / 工艺 / 设计亮点（2-4 句，每句一个差异化点）

Основные особенности:
1. ...
2. ...
3. ...

Характеристики:  ← 只写本 SKU
• Материал: ...
• Размер: ...
• Вес: ...
• Цвет: ...

Способ применения:
...

В комплекте:
• 1 × ...
```

**改写时硬约束**：
- 不编造原稿没有的事实参数（材质/尺寸/容量）——抓不到就写 `[待补]`
- 删尺寸列举（如原稿写 `S, M, L`，问用户：本 listing 是哪一档？或建议拆 SKU）
- 删 Aliexpress 套话（色差 / 手工测量误差）
- 删机翻味重的句子，重写为地道俄语
- 不加禁用营销词
- 字数控制在 500-1500，4-6 段

如果是 `模式=双语对照`，输出两栏：
- 左：改写后俄语
- 右：中文回译（让用户验证语义没飘）

### Step 7: 末尾给"一键复用"提示

```
✏️ 改完后建议：
1. 把新描述粘到 Ozon 后台 → 保存草稿
2. 后台预览检查段落渲染（Ozon 的换行规则较严）
3. 如需同步更新标题/属性，调用 /ozon-card
```

## 输出案例（让模型对齐）

**用户输入**：
```
/ozon-desc 100% абсолютно новый и качественный
Тип: Расческа
Цвет: коричневый
Тип волос: все типы волос
Материал: дерево
Размер: S, M, L
```

**预期输出骨架**：
- 总分 ~2/10，红
- ① 1分（能看出是梳子但太套路）② 0分 ③ 1分（机翻味重）④ 1分（只有参数无用途/卖点/使用方法）⑤ 0分（无段落无编号）
- 雷区：B 机翻命中 / C 尺寸列举命中（S,M,L）
- 改写：按 §2.7 骨架重写，先问用户"实际尺寸是哪一档"再定稿

## 注意事项

1. **规则源单一**：所有判定引用 [ozon/card-requirements.md §2](../../../ozon/card-requirements.md)。规则更新只改那一处。
2. **不要瞎补参数**：用户没给的材质/尺寸/容量，写 `[待补]`，不要根据"通常""一般"凭空填。
3. **机翻判定要给证据**：不要光说"机翻味重"，要指出具体哪句、为什么不地道。
4. **尺寸列举的处理**：发现 `Размер: S, M, L` 这种，**先反问用户**"本 SKU 是哪个尺寸 / 是否要拆多 SKU"，不要默默选一个。
5. **与 `ozon-card` 的边界**：本 skill **只管描述**。要同步生成标题/属性/图片清单走 `/ozon-card`。
6. **课件案例是基准**：评分时心里装着 §2.6 的 ✅Quilin 范例和 ❌野猪鬃梳反例——红线就是后者那些问题。
