# Ozon 自动上架 — 设计交接简报（自包含）

> 给设计/新会话用：不依赖任何历史对话，看这一篇即可接手。

## 1. 一句话

把"跨境卖家把 1688/竞品商品上架到 Ozon(俄罗斯)"这件事做成 **Web 后台一键流程**：自动出俄语文案 + 商品图 + 富文本，合规、避查重，人工确认后发布到 Ozon。

## 2. 现有系统（已在运行，不是从零）

- **栈**：FastAPI(后端) + Vue3/Element Plus(前端) + SQLite(本地)/MySQL(服务器 Docker) + 阿里云 OSS(图床) + Ozon Seller API。
- **已工作**：浏览器插件采集(Ozon/WB/1688)→ 存"草稿(draft)"；草稿编辑器；AI 文案(俄语标题/描述/属性)；多店管理；发布到 Ozon(图自动转 OSS 直链、富文本走属性 11254)；FBS 订单/仓库等。
- **草稿(draft)是核心数据**：所有来源归一成 draft，下游来源无关。图存 `draft.images`，源数据存 `draft.source_raw`。
- **出图能力**：① Agnes 网关；② gpt-image 中转站(模型 `gpt-image-2-2`，支持 create/edit/mask，**edit=传参考图改图**)。引擎由 admin 在设置里切(`ai_image.engine`)。

## 3. 要做的：自动上架，四条路径（按"省事"分层，默认走轻的）

| 路径 | 场景 | 做什么 | 轻重 |
|---|---|---|---|
| **复制** | Ozon 竞品、且源卡允许复制 | 官方 `import-by-sku` 复制成自己的卡 | 最轻，零做图 ✅已实现 |
| **模式1 图片俄化**（最高频） | 已有图/1688 拉的图能用，只是图上是中文 | 只把**图上中文换成俄语**，其余不动 | 轻 |
| **模式2 简单修改** | 想小改某张图 | 换背景/去 logo/去水印/局部改 | 轻 |
| **模式3 完整重做** | 要原创、彻底避查重 | OCR 提取 → 出整套 10 张图 + 文案 + 富文本 | 重 |

## 4. 端到端流程

```
采集 → draft
  ├─ 复制? 试 import-by-sku → 可复制:复制卡 / 不可复制:↓
  ├─ 理解层(跑一次,缓存): OCR(读图上文字) + 角色标注 + 事实抽取
  ├─ 文案(1次LLM,复用事实): 俄语 标题 + 简介(描述+营销) + 标签
  ├─ 图片(gpt-image-2-2 edit,全程传源图):
  │     主图白底 / 卖点 / 尺寸 / 场景 / 富文本底图
  │     文字(俄语)直接让 AI 渲染在图上(数字要人工核对)
  ├─ 富文本 = 把图拼成全宽大图序列(billboard)
  └─ 人工确认 → 发布到 Ozon
```

## 5. 关键决策 + 理由（这些约束塑造设计）

1. **理解用 OCR，不用 vision**：1688 卖点/尺寸都写在图上 → OCR 把图变文本，后面全是便宜的文本调用；vision 太贵。理解只做一次，文案和图片共用。（OCR 用阿里云电商 OCR，和 OSS 同一套凭据，免本地部署。）
2. **图片文字让 gpt-image-2-2 渲染，不用 PIL**：PIL 叠字生硬、和图融不到一起。代价：**AI 渲染精确数字(尺寸/重量)可能写错 → 必须人工 QC 核对数字**。
3. **出图一律 edit 模式 + 传源产品图**：保证产品颜色/结构/接口一致，不变形、不虚构配件。
4. **富文本 = billboard 大图序列**：文字烤在图里(就是上面的卖点图)，所以富文本就是有序大图，无需单独文本块。图链可直接用 OSS 公网直链(已核实 Ozon 接受外链并自动转存；坑：URL 要无签名/不过期、抓取前别删源图)。
5. **分四路径、默认走轻的**：80% 场景是"复制"或"俄化"，不该逼所有人走完整重做。
6. **避查重**：跟卖竞品图时，只换文字图还像竞品→ Ozon 感知哈希可能仍判重；要真避查重得走模式3 重做。模式1 适合"图本来能用(自己的/供应商授权图)"。
7. **合规**：Ozon 图片硬规则(3:4、1024×1536、白/浅底、无中文/水印/logo/价格/二维码、主图无文字)；复制尊重源卡"禁止复制"开关；不保留竞品品牌/水印。

## 7. 需要"设计"的（开放问题，给设计会话）

- **用户操作流(UX)**：四条路径在草稿编辑器里怎么呈现？是一个"智能推荐路径"还是四个明确入口？
- **人工确认环节**：哪些必须人工核(尺寸数字、品牌、待确认参数)？怎么把"AI 待核对"标出来让用户快速确认？
- **图片计划的可视化**：10 张图的方案/进度/重试怎么展示？失败重试 ≤2 的交互。
- **模式1 俄化的交互**：批量俄化时，逐图预览中→俄对照？一键全俄化 vs 逐图确认？
- **富文本预览**：billboard 序列怎么预览、排序、增删。
- **成本可见**：每个 SKU 出图次数/token 给不给用户看。

## 9. 约束速记
- 服务器在国内(阿里云)，国外服务(GitHub 下模型/Docker Hub)受限 → 依赖尽量云端或随镜像。
- 个人/小团队维护 → 简单优先，少加依赖。
- 目标语言俄语；图片 3:4 / 1024×1536；目标平台 Ozon(俄区)。
