# 变体组批量 AI 操作 — 设计

日期：2026-06-25
状态：已与用户确认，待实现

## 背景与目标

同一 `variant_group` 的多个变体草稿会合并成一张 Ozon 多变体卡。目前每个 AI 步骤
（看图理解 / 类别识别 / 生成文案 / 特征填充）只能逐个变体单独点，N 个变体要点 N 遍。
目标：把这些步骤提升到**变体组级别**，一次操作覆盖整组所有变体。

图集计划 / AI 设计并生成 10 张图**不进组级**——每个变体图片不同，仍在单个变体上手动做。

## 步骤的执行口径（per-variant vs 共享）

| 步骤 | 口径 | 说明 |
|---|---|---|
| 看图理解 | 每个变体各跑 | 变体图不同，各自理解 |
| 类别识别 | 跑 1 次 → 自动同步全组 | 合并成一张卡，类别必须一致；已有 `_sync_group_category` |
| 生成文案(标题/简介/标签) | 每个变体各跑 | 用户明确要每个变体各生成各的 |
| 特征填充 | 每个变体各跑 | 区别特征(颜色/尺寸/型号)每变体不同 |

## UI

变体组面板（`.variant-box`）标题行下方新增一排组级按钮：

- 主按钮 **「整组一键跑」**：对所有变体依次跑 看图理解 → 类别 → 文案 → 特征。
- 4 个独立组级按钮：**组·看图理解 / 组·类别 / 组·文案 / 组·特征**，各自对所有变体跑那一步。
- 进度：主按钮 / 各按钮 loading + 文案「文案 3/4…」；跑完用 `ElMessage` 汇总「成功 X / 失败 Y」，失败列出「变体规格 + 步骤」。
- loading 状态存入 store 的 `pendingOps`（按 当前代表草稿 id + 操作名），切走再切回仍转圈。

## 执行架构：前端编排 + 并发池

- **前端编排**，复用现有单草稿接口，对每个变体 id 调用：
  - `api.understand(id)` / `api.recognizeCategory(id)` / `api.aiCopy(id)` / `api.aiFillAttributes(id)`
- **并发池 = 4**（复用 `runAutoImages` 里同款的并发池写法）。
- **类别**：只在**代表变体**(当前 props.draft.id)跑 1 次——`recognize_category` 已自动 `_sync_group_category` 到全组，无需每个变体都跑。
- **看图理解 / 文案 / 特征**：对每个变体并发跑（同时最多 4 个）。
- 为什么前端编排而非后端一个大接口：避免单个超长请求被反代超时（本项目踩过 502），进度直观、单个失败可重试。

## 重试（两级）

1. **网关级**（已上线）：`ai_card.deepseek_chat` 对 429/500/502/503/504 + 连接/超时退避重试最多 3 次。
2. **操作级**（本次新增）：组批量里，对每个变体的每一步整步失败就重试，**最多 3 次**（退避 ~0.8s/2s）。
   - "失败"判定：抛异常 / 返回体含 `error`（如 `aiFillAttributes` 的 `{error}`）/ 类别返回 `matched:false`。
   - 3 次都失败 → 记入失败列表，不中断其它变体。

## 数据一致性（后端，在特征填充 `ai_fill_attributes` 内处理）

1. **型号名称统一**：属性 9048 = 该组 `variant_group`（整组同值，Ozon 才合并成一张卡）。
   现状已优先用 `variant_group`；本次确认并补强：只要 draft 在组内，9048 强制 = `variant_group`。
2. **货号随机**：特征填充时若该变体 `offer_id` 为空 → 生成唯一随机货号 `SP-<8位hex>`；已有的不动。

## 错误与边界

- 单变体单步失败（重试 3 次后）：计入失败列表，继续其它。
- 组只有 1 个变体：组级按钮不显示（`variantSiblings.count <= 1`）。
- 货号唯一性：`SP-` + 随机 8 位 hex，碰撞概率极低；后端生成时可简单校验本组内不重复。

## 测试

- 后端：`ai_fill_attributes` 对组内 draft 强制 9048=variant_group + 空 offer_id 生成 `SP-` 的单测。
- 前端：并发池 + 操作级重试为纯逻辑，可抽函数做轻量验证（手动为主）。

## 不做（YAGNI）

- 不做后端单接口批量编排（超时风险）。
- 不做图集/出图的组级（图片按变体独立）。
- 不做变体行内逐步状态细粒度展示（先做整体进度 + 汇总；不够再加）。
